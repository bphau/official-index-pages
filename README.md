# official-index-pages

## 项目简介

本仓库用于归档和发布多个独立的 HTML 页面。这些页面可以作为独立的索引、引导页或基础展示页面使用。仓库本身不绑定任何特定域名或网站，仅作为页面资源的集中存放与版本管理。

## 目录说明

```
├── pages/            # 存放所有归档的 HTML 页面
├── assets/           # 页面引用的公共资源（CSS、JS、图片等）
├── .gitignore        # 忽略文件配置
└── README.md         # 本文件
```

- `pages/`：每个 HTML 页面以独立文件形式存放，文件名即页面标识。
- `assets/`：存放多个页面可能共用的样式、脚本或媒体文件，便于统一维护。

## 页面归档说明

- 所有页面均为纯前端 HTML 文件，不依赖后端服务。
- 页面内容保持简洁、通用，可直接在浏览器中打开使用。
- 归档的页面可能包含基础导航、信息展示、链接集合等常见功能模块。
- 每个页面均经过基本可用性检查，确保结构清晰、无明显错误。

## 维护说明

- 新增页面请放入 `pages/` 目录，并保持命名规范（如使用小写字母和连字符）。
- 若页面需要引用公共资源，请优先使用 `assets/` 中的文件，避免重复引入。
- 更新或移除页面时，请同步更新相关引用或说明。
- 欢迎通过 Issue 或 Pull Request 提出改进建议或提交新的页面归档。

## 使用建议

- 可通过 GitHub Pages 或其他静态托管服务将本仓库内容部署为可访问的页面集合。
- 也可以直接克隆仓库到本地，在浏览器中打开 `pages/` 下的 HTML 文件查看效果。
- 仓库内容不涉及任何特定商业或推广信息，适合作为通用页面存档使用。
